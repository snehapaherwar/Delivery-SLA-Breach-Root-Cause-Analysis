# Delivery SLA Breach Analysis

![Dashboard](visuals/sla_dashboard.png)

## Overview

Timely order delivery is a critical factor influencing customer satisfaction and operational efficiency. This project analyzes delivery performance to identify SLA (Service Level Agreement) breaches, uncover operational bottlenecks, and recommend actions to improve on-time delivery rates.

The analysis focuses on measuring delivery compliance, understanding delay drivers, and quantifying the impact of SLA breaches across regions, delivery partners, and order categories.

---

## Business Problem

The business observed an increase in delayed deliveries, leading to:

- Reduced customer satisfaction
- Increased customer complaints
- Higher refund and compensation costs
- Operational inefficiencies

The objective was to identify the primary causes of SLA breaches and recommend strategies to improve delivery performance.

---

## Project Objectives

- Measure SLA compliance across all deliveries
- Identify orders that breached delivery commitments
- Analyze breach patterns by region, city, and delivery partner
- Determine root causes of delays
- Recommend operational improvements

---

## Dataset

The dataset contains delivery-level information including order details, promised delivery timelines, and actual delivery timestamps.

### Key Fields

| Field | Description |
|---------|-------------|
| Order ID | Unique order identifier |
| Customer ID | Unique customer identifier |
| Order Date | Date order was placed |
| Dispatch Date | Date order was dispatched |
| Promised Delivery Date | SLA commitment date |
| Actual Delivery Date | Actual delivery completion date |
| Delivery Partner | Logistics provider |
| City | Delivery location |
| Region | Geographic region |
| Order Value | Revenue generated by order |

---

## Methodology

### 1. Data Cleaning

- Removed duplicate records
- Validated timestamp consistency
- Standardized delivery partner and location names

### 2. SLA Calculation

Calculated delivery turnaround time:

```sql
DATEDIFF(day, order_date, actual_delivery_date)
```

Calculated SLA status:

```sql
CASE
    WHEN actual_delivery_date <= promised_delivery_date
         THEN 'Within SLA'
    ELSE 'Breached SLA'
END
```

---

### Delivery Partner Analysis

- Partner C contributed the highest number of SLA breaches.
- Partner A maintained the highest on-time delivery rate.
- Significant performance variation existed across logistics providers.

---

### Regional Analysis

- Metropolitan regions showed the strongest SLA compliance.
- Tier-2 and Tier-3 cities experienced higher delay rates.
- Certain fulfillment centers consistently generated delayed shipments.
